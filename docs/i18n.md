# Internationalization (i18n)

Kanso ships a small, opt-in i18n surface. Two injection tokens + a handful of `Intl`-backed helpers cover the common cases (date / time formatting, weekday names, button labels) without dragging in a translation framework.

The package is `@kanso-protocol/ui/i18n`. Components that need locale-aware behavior depend on it as a peer; consumers provide the tokens at the app root.

## The two tokens

### `KP_LOCALE` ÔÇö BCP-47 language tag

Drives every Intl-backed call: month names, weekday labels, time formatting, AM/PM markers.

Default factory: `navigator.language` if available, else `'en-US'`.

```ts
import { KP_LOCALE } from '@kanso-protocol/ui/i18n';

bootstrapApplication(AppComponent, {
  providers: [
    { provide: KP_LOCALE, useValue: 'fr-FR' },
  ],
});
```

### `KP_STRINGS` ÔÇö static UI labels

A `Partial<KpLocaleStrings>` registry. Override only the keys you need; missing keys fall through to English defaults.


#### Validation messages (`validation` sub-key)

Form-field error copy lives under the nested `validation` key of `KP_STRINGS` ÔÇö a dictionary keyed by the Angular validator name (`required`, `email`, `pattern`, `minlength`, `maxlength`, `min`, `max`, plus any custom validator). Each value is either a plain string or a function of the error payload:

```ts
import { KP_STRINGS } from '@kanso-protocol/ui/i18n';

{
  provide: KP_STRINGS,
  useValue: {
    validation: {
      required:  'đ×đ▒ĐĆđĚđ░ĐéđÁđ╗ĐîđŻđżđÁ đ┐đżđ╗đÁ',
      email:     'đĺđ▓đÁđ┤đŞĐéđÁ đ║đżĐÇĐÇđÁđ║ĐéđŻĐőđ╣ email',
      minlength: ({ requiredLength }) => `đťđŞđŻđŞđ╝Đâđ╝ ${requiredLength} ĐüđŞđ╝đ▓đżđ╗đżđ▓`,
    },
  },
}
```

`<kp-form-field>` re-bases every key on the English defaults, so a partial override is fine ÔÇö validators you don't list keep their built-in message. Per-field overrides via the component's `[errors]` input still win over the registry.

##### Back-compat: the deprecated `KP_VALIDATION_MESSAGES` token

Validation messages used to live in a standalone `KP_VALIDATION_MESSAGES` token exported from `@kanso-protocol/ui/form-field`. That token is now **deprecated but still honored** ÔÇö its entries are merged on top of `KP_STRINGS.validation`, so existing apps keep working without changes. Resolution order, highest wins:

```
per-field [errors]  Ôćĺ  KP_VALIDATION_MESSAGES  Ôćĺ  KP_STRINGS.validation  Ôćĺ  built-in English defaults
```

Prefer `KP_STRINGS.validation` for new code so all localizable copy sits behind a single token; `KP_VALIDATION_MESSAGES` will be removed in a future major.

## The Intl helpers

Use these inside components instead of hand-rolling `Intl.DateTimeFormat`. They cache formatters per `(locale, options)` pair so calling them in a render loop is cheap.

```ts
import {
  kpFormatDate,
  kpFormatTime,
  kpMonthNames,
  kpWeekdayNames,
  kpDayPeriod,
  injectKpLocale,
} from '@kanso-protocol/ui/i18n';

const locale = injectKpLocale();

kpFormatDate(new Date(), locale, { dateStyle: 'medium' });  // "May 4, 2026"
kpFormatTime(new Date(), locale);                            // "3:45 PM" or "15:45"
kpMonthNames(locale, 'long');                                // ['January', ..., 'December']
kpWeekdayNames(locale, 1, 'narrow');                         // ['M', 'T', 'W', 'T', 'F', 'S', 'S']
kpDayPeriod(15, 'en-US');                                    // 'PM' (or '' for 24h locales)
```

## Usage in a Kanso component

```ts
import { Component, inject } from '@angular/core';
import { injectKpLocale, injectKpStrings, kpFormatDate } from '@kanso-protocol/ui/i18n';

@Component({
  template: `
    <button [attr.aria-label]="strings.clear">├Ś</button>
    <span>{{ today }}</span>
  `,
})
export class MyComponent {
  private locale = injectKpLocale();
  protected strings = injectKpStrings();
  protected today = kpFormatDate(new Date(), this.locale);
}
```

| Component | Status | What's localized |
|-----------|--------|------------------|
| `datepicker` | Ôťô | Month names (long + short), weekday labels, date format, preset labels (Today / Yesterday / This week / etc.), ARIA labels for clear / prev / next |
| `timepicker` | Ôťô | 12h/24h **auto-detected** from `KP_LOCALE` (override via `[format]`), AM/PM markers via `kpDayPeriod()`, column labels (`Hour` / `Min` / `Sec` / `AM/PM`), `Now` / `Cancel` / `Confirm` footer, `Clear time` / `Choose time` ARIA |
| `file-upload` | Ôťô | `dropzoneTitle`, function-typed `dropzoneRemove(filename)`, `dropzoneUploaded` |

| `search-bar` | Ôťô | `searchPlaceholder`, `commandPaletteHint` |
| `pagination` | Ôťô | Host `aria-label` (`paginationLabel`), prev/next labels + page-cell ARIA (function-typed `paginationGotoPage(n)` / `paginationCurrentPage(n)`), items-per-page label, function-typed `paginationShowing(from, to, total)` |

## Why not full message catalogs?

A full message-catalog framework (Angular i18n, ngx-translate, transloco) is the right tool when you have hundreds of strings and a translation pipeline. For a design system where each component owns ~5 strings and consumers usually translate at the app level anyway, a small DI-backed registry is enough.

If a Kanso consumer wants to plug into a real catalog, they can ÔÇö wrap their catalog's lookup in a `Partial<KpLocaleStrings>` factory and provide it via `KP_STRINGS`.

### Shipped

- **Validation messages folded into `KP_STRINGS`** ÔÇö copy now lives under `KP_STRINGS.validation`; the standalone `KP_VALIDATION_MESSAGES` token is deprecated but still honored (merged on top). See the [`validation` sub-key](#validation-messages-validation-sub-key) section above.
- **12h/24h auto-detection in `timepicker`** ÔÇö the effective clock convention is derived from `KP_LOCALE` (via `Intl.DateTimeFormat(locale, { hour: 'numeric' })` day-period detection) and used as the default. The `[format]` input still forces `'12h'` / `'24h'` explicitly; its default is now `'auto'`.
