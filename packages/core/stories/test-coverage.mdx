---

**Что именно покрывают тесты** (минимальный бар, применяемый к каждой спеке):

1. **Rendering** — компонент монтируется без ошибок
2. **Inputs → DOM** — `@Input()` отражается в классе/атрибуте/дочернем DOM
3. **Accessibility** — `role`, `aria-*`, `tabindex` соответствуют контракту
4. **Events** — `@Output()` стреляет при документированных триггерах; заблокированные состояния (disabled/loading) глушат эмиты
5. **ControlValueAccessor** — `writeValue` + `registerOnChange` для form-компонентов

Подробности паттернов написания — в [CONTRIBUTING.md](https://github.com/GregNBlack/kanso-protocol/blob/main/CONTRIBUTING.md#testing).
